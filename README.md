# key-pulse-view

An education KPI dashboard built with Vite, React, TypeScript, Tailwind, and shadcn/ui. Eight tabbed views surface metrics across student journey, equity, teacher quality, and institutional efficiency.

> All data is mock data, kept under `src/data/`. There is no backend yet.

## Quick start

```bash
bun install
bun dev          # http://localhost:8080
```

## Scripts

| Command                 | What it does                          |
| ----------------------- | ------------------------------------- |
| `bun dev`               | Start the Vite dev server (port 8080) |
| `bun run build`         | Production build                      |
| `bun run build:dev`     | Build with development mode flags     |
| `bun run preview`       | Preview the production build locally  |
| `bun run lint`          | Run ESLint                            |
| `bun test`              | Run unit tests once (Vitest)          |
| `bun run test:watch`    | Vitest in watch mode                  |
| `bunx playwright test`  | Run end-to-end tests                  |

## Tabs

The dashboard is a single page with these views (see `src/pages/Index.tsx`):

| Tab                       | Component                                       | Data source                     |
| ------------------------- | ----------------------------------------------- | ------------------------------- |
| Overview                  | `dashboard/landing/StrategicLanding.tsx`        | `data/kpiData.tsx`              |
| Executive Summary         | `dashboard/ExecutiveSummary.tsx`                | `data/kpiData.tsx`              |
| Student Journey           | `dashboard/student/StudentJourneyView.tsx`      | `data/studentJourneyData.ts`    |
| Student Profile           | `dashboard/profile/StudentProfileView.tsx`      | `data/studentProfileData.ts`    |
| Access & Equity           | `dashboard/equity/EquityView.tsx`               | `data/equityData.ts`            |
| Teacher Excellence        | `dashboard/teacher/TeacherView.tsx`             | `data/teacherData.ts`           |
| Quality Assurance         | `dashboard/quality/QualityView.tsx`             | `data/qualityData.ts`           |
| Institutional Efficiency  | `dashboard/efficiency/EfficiencyView.tsx`       | `data/efficiencyData.ts`        |

To add a tab, extend `DashboardTab` in `src/components/dashboard/DashboardTabs.tsx`, add a `tabs` entry, and add a `case` in the `switch` inside `src/pages/Index.tsx`.

## Tech stack

- **Vite 5** + **React 18** + **TypeScript** (SWC)
- **Tailwind CSS 3** + **shadcn/ui** (Radix primitives in `src/components/ui/`)
- **TanStack Query**, **React Router 6**, **React Hook Form** + **Zod**
- **Recharts** charts, **lucide-react** icons, **next-themes**
- **Vitest** + **Testing Library** (jsdom) for unit tests, **Playwright** for E2E

## Project layout

```
src/
  App.tsx              router + providers
  main.tsx             entry point
  pages/               Index, NotFound
  components/
    dashboard/         feature components (one folder per tab)
    ui/                shadcn primitives — generated, don't hand-edit
  data/                mock KPI datasets
  hooks/               use-mobile, use-toast
  lib/utils.ts         cn() helper
  test/setup.ts        vitest setup
```

Path alias: `@/*` → `src/*`.

## shadcn components

Primitives in `src/components/ui/` are generated by the shadcn CLI (config in `components.json`, slate base, CSS variables enabled). To add a new one:

```bash
bunx shadcn@latest add <component>
```

Don't hand-edit existing primitives — re-run the CLI if you need updates.
